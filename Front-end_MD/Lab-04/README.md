## Лабораторная 4.1: Обработка событий и валидация форм

### Подход к обработке событий
- **Контролируемые компоненты**: Все поля формы используют проп `value` связанный с состоянием React и обработчики `onChange` для обновления состояния (паттерн из Главы 4).
- **Синтетические события**: Использование кросс-браузерной обертки событий React (`e.preventDefault()`, `e.target.value`).
- **Встроенные обработчики**: Стрелочные функции в JSX для немедленного обновления состояния и валидации.
- **Стратегия валидации**: Валидация в реальном времени при изменении поля + финальная валидация при отправке.

## Лабораторная 4.2: Рефакторинг компонентов

### Подход к рефакторингу
Следуя паттернам Главы 5:
1. **Монолитный → Модульный**: Разбиение ArticleManager на меньшие компоненты
2. **Обязанности компонентов**:
   - `ArticleManager`: Контейнер, управляет состоянием статей и обработчиками
   - `AddArticle`: Презентационный, поля ввода для новых статей
   - `ArticleList`: Контейнер, рендерит список ArticleItem
   - `ArticleItem`: Презентационный, управляет собственным состоянием `isOpened` локально
3. **Управление состоянием**: Перенос переключения отображения из родителя в ArticleItem (лучшая производительность)
4. **Render Props**: Опциональный паттерн для подмены компонентов без прямых зависимостей

### Файлы
- `src/components/ArticleManager.jsx` - Контейнерный компонент
- `src/components/AddArticle.jsx` - Форма добавления статьи
- `src/components/ArticleList.jsx` - Контейнер списка статей
- `src/components/ArticleItem.jsx` - Отдельная статья с переключением
- `src/App.jsx` - Главное приложение демонстрирующее обе лабораторные
